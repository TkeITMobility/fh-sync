DatasetClientsCleaner(opts, retentionPeriod, checkFrequency)

Constructor for DatasetClientsCleaner. A DatasetClientsCleaner will check the status of all the dataset clients in the db, and if any of them hasn't been accessed for the past time, it will remove them from the db to save space. By default, it will remove dataset clients that haven't been active for 1 day and it will run the check every hour. Developer can change those parameters via the configuration.

Name Type Description

checkFrequency String how often the checks should be running. Default is '1h'.

removeAppliedUpdates(clientRecords, localDatasetClient, appliedUpdates)

We have some updates that are applied to the backed since last sync run. However, those updates are not in the localDatasetClient yet (they will in after next sync run). But those updates are applied on the client already. This means when we compute the delta between the client records and the localDatasetClient, those changes will be reverted from the client. To avoid that, we loop through those updates and remove them both from the clientRecords and localDatasetClient. This way there will be no delta for those records and the client will keep there records.
Parameters:
Name Type Description
clientRecords any
localDatasetClient any
appliedUpdates any

removePendingChanges(clientRecordds, localDatasetClient, pendingChanges)

We have some pending changes sent in by the client and waiting in the queue. They are not processed yet so the backend or our localDataset don't have those changes. We don't know if those changes will be applied or not. But they are already applied on the client which means if we compute the delta as it is, those changes will be reverted. To avoid that, we will remove those changes from both the client and the localDataset so that the client will keep them for now.
Parameters:
Name Type Description
clientRecordds any
localDatasetClient any
pendingChanges any
